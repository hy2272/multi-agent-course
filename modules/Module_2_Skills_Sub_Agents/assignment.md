Sprint Zero Prototype

Project • Submit by FRI. JUN 12

Overview
In this assignment, you will work with a product called SprintZero. The product description, workflow document, slide deck, and GitHub codebase link are provided to you for reference. These materials describe the product vision, workflow, core functionality, and current technical foundation.

Your goal is to study the product carefully and create a meaningful deliverable based on your role: Product Manager or Developer.

You may choose the track that best matches your background or learning goal.





Track 1: Product Manager Assignment


Objective
Imagine this is a product your team has built. The product concept, workflow, slide deck, and supporting materials have already been created by your team. Now, as the Product Manager, your task is to create a website for the product that can be used for MVP marketing, stakeholder communication, early user acquisition, or product validation.

The website should clearly communicate what the product does, who it is for, what problem it solves, and why users or customers should care.



What You Need to Build
Create a website for the product. This can be a landing page, MVP marketing website, product demo site, or early go-to-market website.

Your website should include:

Product Name and Clear Hero Section
Introduce the product with a strong headline, short value proposition, and clear call-to-action.

Problem Statement
Explain the problem this product is solving.

Target Users or Customers
Describe who the product is built for.

Product Overview
Explain what the product does in simple, clear language.

Key Features
Highlight the most important features or capabilities of the product.

Workflow or How It Works
Use the provided workflow document and slide deck to explain how the product works.

Benefits and Value Proposition
Explain why this product is useful and what value it creates.

Call-to-Action
Include a CTA such as “Request a Demo,” “Join the Waitlist,” “Try the MVP,” or another appropriate action.

Design and User Experience
The website should be clean, understandable, and suitable for presenting the product to early users, stakeholders, or investors.



Optional Enhancements
You may also include:

Pricing section

FAQ section

Testimonials or placeholder customer quotes

Demo video placeholder

Screenshots or mockups

Contact form

Competitive positioning

Product roadmap

Blog or documentation section



Submission
Submit your completed website as a .zip folder.

Your zip folder should include:

All website files

Any assets used, such as images, icons, or logos

A short README file explaining how to open or run the website

Any assumptions you made while creating the website





Track 2: Developer Assignment


Objective
As a developer, your task is to review all provided product materials, understand the product vision and workflow, examine the GitHub/codebase provided, and implement at least one meaningful component from Sprint Zero.

Sprint Zero refers to the early technical foundation of the product. Your goal is to build one core piece that helps move the product from concept toward a working implementation.



What You Need to Build
Implement at least one core component of the product.

This could include, but is not limited to:

Frontend Component
Build a key user interface screen, form, dashboard, workflow page, or interactive component.

Backend Component
Implement an API endpoint, database logic, authentication flow, processing function, or service layer.

Workflow Implementation
Convert part of the workflow document into working functionality.

Data Processing or AI Component
Implement a model integration, prompt pipeline, data parser, recommendation logic, search function, or automation step.

Codebase Improvement
Improve, refactor, optimize, or extend part of the existing codebase.

Prototype Feature
Build a functional prototype of one important product feature.



You are required to build at least one component. You are welcome to build more than one component or attempt a larger implementation if you want.



Expectations
Your implementation should show that you have:

Reviewed the product description, workflow document, slide deck, and codebase

Understood the purpose of the product

Identified at least one technically meaningful piece to implement

Written clean, understandable code

Added documentation or comments where useful

Made reasonable assumptions when details were missing



Improvement and Optimization Suggestions
As part of your submission, include a short section explaining any improvement opportunities you noticed.

You may mention:

Product improvement ideas

Technical architecture improvements

UX improvements

Workflow simplifications

Code optimization opportunities

Scalability concerns

Missing features

Risks or edge cases

Suggestions for future development

This section does not need to be long, but it should show thoughtful review of the product and codebase.



Submission
Submit your completed work as a .zip folder.

Your zip folder should include:

Your implemented code

Any modified or added files

A README file explaining:

What you built

Why you chose this component

How to run or test it

Any assumptions you made

Any improvements or optimizations you recommend

Screenshots or demo video link, if applicable
